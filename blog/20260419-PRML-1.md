## Overview

The goal of pattern recognition is to **<font color="red">predict future observations based on observed data</font>**.
Polynomial curve fitting assumes that observed data are generated by some function and then **<font color="red">approximates that function with a polynomial</font>**.

Here is an example.
Suppose we obtain data such as 0.07 when x=0, 0.32 when x=0.125, 0.19 when x=0.25, and so on, and can plot the points as follows.

![graph1-1.jpg](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/374669/e99ea2e7-3578-3d44-99bd-937caa68157b.jpeg)

If we assume these points are generated by **some function**, one candidate is <font color="red">"f(x)=x"</font>.

Now, to check whether the function "f(x)=x" actually approximates this data well, let us plot it.

![graph1-2.jpg](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/374669/90f4ae8d-c927-ee13-f3b1-e1fd97b6fbe9.jpeg)

You can see that most data points are close to **"f(x)=x"**.
Here, note that data contain noise.
The true value possessed by the target is the **<font color="red">true value</font>**, but most values we observe include some noise and become **<font color="red">true value + noise</font>**.

Therefore, if we take noise into account, the function **"f(x)=x"** can be regarded as a reasonably good prediction.

In fact, **this data was generated by adding noise to "f(x)=x"**.
In other words, **we successfully predicted the function that generated the data**.

In this example, we tried approximation by a first-order function, but **real-world data is rarely this linear and simple**, so appropriate curve fitting by polynomials is required.


## Theory

In the overview, we discussed the case where generated data can be approximated by **"f(x)=x"**.

However, that function was **chosen by guesswork**.
It was not selected from a mathematically grounded slope and intercept; it just happened to fit.
That does not guarantee it will work for another observed dataset.

So, we introduce **a criterion (objective metric) and approximate the function mathematically based on that criterion**.

A common example is **<font color="red">"minimization of squared error"</font>** between data points and the approximating function.
When the error (residual) is large, the squared error is large; when the residual is small, the squared error is small.
Thus, **minimizing squared error is an appropriate indicator that the approximating function is correct**.

![graph1-3.jpg](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/374669/6e94abcf-5e9d-64db-6b0e-1f23e86a7c52.jpeg)

Below, based on minimization of squared error, we explain how to design an approximating function.

![graph1-4.jpg](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/374669/3ab8a729-b2d9-fb7e-1d93-1c6269d12b6a.jpeg)

Let the approximating first-order function be "f(x)=ax+b", input data be x, observed data corresponding to x be y, number of observations be m, and sum-of-squares error be E. Then E is expressed as:

```math
E=\sum_{i=1}^{m}\{y_i-f(x_i)\}^2\,...(1)
```

What we ultimately want is **the parameters (a, b) of f(x) that minimize E**.
So first, we consider **minimizing E**.

Substituting f(x) into Eq. (1) gives Eq. (2):

```math
E=\sum_{i=1}^{m}(y_i-ax_i-b)^2\,...(2)
```

Here, input data x and observed data y are known, and constants a and b are unknown.
Therefore, the sum-of-squares error E is a two-variable function E(a, b).

To simplify, first fix b (b=0) and assume E is a function of a only, E(a).
Then we can draw the following graph.

![graph1-5.jpg](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/374669/ffdbdd90-26da-0740-95e0-340171337ed9.jpeg)

From the graph, E(a) is a **<font color="red">convex quadratic function</font>**.
In other words, **<font color="red">the vertex of the graph (local minimum) is the minimum of E(a)</font>**.

We can also show that E(a) is convex quadratic mathematically:

```math
\begin{eqnarray}
E &=& \sum_{i=1}^{m}(y_i-ax_i-b)^2 \\
&=& \sum_{i=1}^{m}\{x_i^2a^2+2x_i(b-y_i)a+(b-y_i)^2\} \\
&=& (x_1^2+x_2^2+\cdots+x_m^2)a^2+\sum_{i=1}^{m}\{2x_i(b-y_i)a+(b-y_i)^2\}\,...(3)
\end{eqnarray}
```

From Eq. (3), the coefficient of a^2 is a sum of squares of real numbers, so **the coefficient of a^2 is positive**.
Thus, E(a) is a convex quadratic function.

We now know that when E is E(a), **<font color="red">the vertex (local minimum) is the minimum of E(a)</font>**.
What about when E is E(b), a function of b? The same applies.

```math
\begin{eqnarray}
E &=& \sum_{i=1}^{m}(y_i-ax_i-b)^2 \\
&=& \sum_{i=1}^{m}\{b^2+2(x_ia-y_i)b+(x_ia-y_i)^2\} \\
&=& mb^2+\sum_{i=1}^{m}\{2(x_ia-y_i)b+(x_ia-y_i)^2\}\,...(4)
\end{eqnarray}
```

From Eq. (4), the coefficient of b^2 is m (>0), so E(b) is also a convex quadratic function.
Therefore, when E is E(b), **<font color="red">the vertex (local minimum) is the minimum of E(b)</font>**.

Our original goal was to find the minimum of E(a, b), a two-variable function.
Since E(a, b) is convex quadratic in each variable a and b, **the graph globally has a vertex (local minimum), and that is the minimum of E(a, b)**.

![graph1-6.gif](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/374669/fea8ec07-99d0-b8d0-855a-c63f1db9cc0f.gif)

In short, **<font color="red">finding the minimum of E(a, b) = finding the vertex (local minimum) of E(a, b)</font>**.

So here we focus on **<font color="red">finding the local minimum of E(a, b)</font>**.

A local minimum of a multivariable function exists where derivatives with respect to each variable are zero.
That is, at the point where the partial derivative of E(a, b) with respect to a and with respect to b are both zero.

This can be written as Eqs. (5) and (6):

```math
E = \sum_{i=1}^{m}(y_i-ax_i-b)^2 \\
\left\{\begin{array}{ll}
\frac{\partial E}{\partial a} &=& \sum_{i=1}^{m}-2(y_i-ax_i-b)x_i &=& 0\,...(5) \\
\frac{\partial E}{\partial b} &=& \sum_{i=1}^{m}-2(y_i-ax_i-b) &=& 0\,...(6)
\end{array}\right.
```

Furthermore, by transforming Eqs. (5) and (6):

```math

\left\{\begin{array}{ll}
(\sum_{i=1}^{m}x_i^2)a &+&(\sum_{i=1}^{m}x_i)b &=& \sum_{i=1}^{m}y_ix_i \\
(\sum_{i=1}^{m}x_i^2)a &+&mb &=& \sum_{i=1}^{m}y_i
\end{array}\right.
```

In matrix form:

```math
\begin{eqnarray}

\begin{pmatrix}
\sum_{i=1}^{m}x_i^2 & \sum_{i=1}^{m}x_i \\
\sum_{i=1}^{m}x_i & m
\end{pmatrix}

\begin{pmatrix}
a \\
b
\end{pmatrix}
&=&
\begin{pmatrix}
\sum_{i=1}^{m}y_ix_i \\
\sum_{i=1}^{m}y_i
\end{pmatrix} \\


\begin{pmatrix}
x_1 & x_2 & \cdots & x_m \\
1 & 1 & \cdots & 1
\end{pmatrix}

\begin{pmatrix}
x_1 & 1 \\
x_2 & 1 \\
\vdots & \vdots \\
x_m & 1
\end{pmatrix}

\begin{pmatrix}
a \\
b
\end{pmatrix}
&=&
\begin{pmatrix}
x_1 & x_2 & \cdots & x_m \\
1 & 1 & \cdots & 1
\end{pmatrix}

\begin{pmatrix}
y_1 \\
y_2 \\
\vdots \\
y_m
\end{pmatrix}

\end{eqnarray}
```

Here, define

```math
\boldsymbol{X} = \begin{pmatrix}
x_1 & 1 \\
x_2 & 1 \\
\vdots & \vdots \\
x_m & 1
\end{pmatrix}
,
\boldsymbol{w} = \begin{pmatrix}
a \\
b
\end{pmatrix}
,
\boldsymbol{y} = \begin{pmatrix}
y_1 \\
y_2 \\
\vdots \\
y_m
\end{pmatrix}
```

then:

```math
\boldsymbol{X}^T \boldsymbol{X} \boldsymbol{w} = \boldsymbol{X}^T \boldsymbol{y}\,...(7)
```

Also, X^T×X is a square matrix regardless of the number of input data points (assuming it is regular), so it has an inverse (X^T×X)^-1.
Therefore, multiplying both sides of Eq. (7) from the left by (X^T×X)^-1 gives Eq. (8):

```math
\boldsymbol{w} = (\boldsymbol{X}^T \boldsymbol{X})^{-1} \boldsymbol{X}^T \boldsymbol{y}\,...(8)
```

Now we only need to substitute values into the right-hand side to obtain parameters a and b of the first-order function that minimize E(a, b).

This method of obtaining an approximating function using residual sum of squares is called the **<font color="red">least squares method</font>**.<br><br>

Now, let us apply Eq. (8) to the previous data.
Then we obtain the following solution:

```math
\begin{eqnarray}

\begin{pmatrix}
a \\
b
\end{pmatrix}
&=&
\begin{pmatrix}
\begin{pmatrix}
0.000 & 0.125 & \cdots & 1.000 \\
1 & 1 & \cdots & 1
\end{pmatrix}
\begin{pmatrix}
0.000 & 1 \\
0.125 & 1 \\
\vdots & 1 \\
1.000 & 1
\end{pmatrix}
\end{pmatrix}
^{-1}

\begin{pmatrix}
0.000 & 0.125 & \cdots & 1.000 \\
1 & 1 & \cdots & 1
\end{pmatrix}

\begin{pmatrix}
0.099 \\
-0.077 \\
\vdots \\
1.050
\end{pmatrix} \\

&=&

\begin{pmatrix}
1.143 \\
-0.045
\end{pmatrix}

\end{eqnarray}
```

Hence, the approximating function is:

```math
f(x) = 1.143x - 0.045
```

You can see that there is very little error.

### Approximation with an n-th order function

Now, generalizing the first-order case, let us try approximation with an n-th order function.

```math
f(x) = w_0 + w_1x + w_2x^2 + \cdots + w_nx^n = \sum_{j=0}^{n}w_jx^j
```

The sum-of-squares error E between observed data y and f(x) is:

```math
\begin{eqnarray}

E &=& \sum_{i=1}^{m}(y_i-w_0-w_1x_i-w_2x_i^2-\cdots-w_nx_i^n)^2 \\
&=& \sum_{i=1}^{m}(x_i^{2l} w_l^2 + Bw_l + C) \\
&=& (x_1^{2l} + x_2^{2l} + \cdots + x_m^{2l})w_l^2 + \sum_{i=1}^{m}(Bw_l + C) \\

\end{eqnarray}
```

Here, l (0≦l≦n, integer) is an index representing an arbitrary coefficient w, and B or C are constants that do not include w_l.
As seen in this formula, x^2l is positive, so the coefficient of w_l^2 (their sum) is positive.

From this, when E is viewed as E(w_l), **<font color="red">the vertex (local minimum) of the graph is the minimum of E(w_l)</font>**, and this **applies to all w**.
Therefore, **globally, the graph has a vertex (local minimum), and that is the minimum of E(w)**.

Now we can obtain the approximating function by least squares.

As in the first-order case, we find the point where partial derivatives are zero and determine w.

```math
\left\{\begin{array}{ll}

\frac{\partial E}{\partial w_0} &=& \sum_{i=1}^{m}-2(y_i-w_0-w_1x_i-w_2x_i^2-\cdots-w_nx_i^n) &=& 0 \\

\frac{\partial E}{\partial w_1} &=& \sum_{i=1}^{m}-2(y_i-w_0-w_1x_i-w_2x_i^2-\cdots-w_nx_i^n)x_i &=& 0 \\

\frac{\partial E}{\partial w_2} &=& \sum_{i=1}^{m}-2(y_i-w_0-w_1x_i-w_2x_i^2-\cdots-w_nx_i^n)x_i^2 &=& 0 \\

&& 　　　　　　　　\vdots \\

\frac{\partial E}{\partial w_n} &=& \sum_{i=1}^{m}-2(y_i-w_0-w_1x_i-w_2x_i^2-\cdots-w_nx_i^n)x_i^n &=& 0

\end{array}\right.
```

```math
\begin{eqnarray}

\sum_{i=1}^{m}w_0 + \sum_{i=1}^{m}w_1x_i + \sum_{i=1}^{m}w_2x_i^2 + \cdots + \sum_{i=1}^{m}w_nx_i^n &=& \sum_{i=1}^{m}y_i \\

\sum_{i=1}^{m}w_0x_i + \sum_{i=1}^{m}w_1x_i^2 + \sum_{i=1}^{m}w_2x_i^3 + \cdots + \sum_{i=1}^{m}w_nx_i^{n+1} &=& \sum_{i=1}^{m}y_ix_i \\

\sum_{i=1}^{m}w_0x_i^2 + \sum_{i=1}^{m}w_1x_i^3 + \sum_{i=1}^{m}w_2x_i^4 + \cdots + \sum_{i=1}^{m}w_nx_i^{n+2} &=& \sum_{i=1}^{m}y_ix_i^2 \\

\vdots　　　　　　　　　　 \\

\sum_{i=1}^{m}w_0x_i^n + \sum_{i=1}^{m}w_1x_i^{n+1} + \sum_{i=1}^{m}w_2x_i^{n+2} + \cdots + \sum_{i=1}^{m}w_nx_i^{2n} &=& \sum_{i=1}^{m}y_ix_i^n

\end{eqnarray}
```

In matrix form:

```math
\begin{eqnarray}

\begin{pmatrix}
m & \sum_{i=1}^{m}x_i & \sum_{i=1}^{m}x_i^2 & \cdots & \sum_{i=1}^{m}x_i^{n} \\
\sum_{i=1}^{m}x_i & \sum_{i=1}^{m}x_i^2 & \sum_{i=1}^{m}x_i^3 & \cdots & \sum_{i=1}^{m}x_i^{n+1} \\
\sum_{i=1}^{m}x_i^2 & \sum_{i=1}^{m}x_i^3 & \sum_{i=1}^{m}x_i^4 & \cdots & \sum_{i=1}^{m}x_i^{n+2} \\
\vdots & \vdots & \vdots & \ddots & \vdots \\
\sum_{i=1}^{m}x_i^{n} & \sum_{i=1}^{m}x_i^{n+1} & \sum_{i=1}^{m}x_i^{n+2} & \cdots & \sum_{i=1}^{m}x_i^{2n} \\
\end{pmatrix}

\begin{pmatrix}
w_0 \\
w_1 \\
w_2 \\
\vdots \\
w_n
\end{pmatrix}
&=&
\begin{pmatrix}
\sum_{i=1}^{m}y_i \\
\sum_{i=1}^{m}y_ix_i \\
\sum_{i=1}^{m}y_ix_i^2 \\
\vdots \\
\sum_{i=1}^{m}y_ix_i^n
\end{pmatrix} \\


\begin{pmatrix}
1 & 1 & 1 & \cdots & 1 \\
x_1 & x_2 & x_3 & \cdots & x_m \\
x_1^2 & x_2^2 & x_3^2 & \cdots & x_m^2 \\
\vdots & \vdots & \vdots & \ddots & \vdots \\
x_1^n & x_2^n & x_3^n & \cdots & x_m^n
\end{pmatrix}

\begin{pmatrix}
1 & x_1 & x_1^2 & \cdots & x_1^n \\
1 & x_2 & x_2^2 & \cdots & x_2^n \\
1 & x_3 & x_3^2 & \cdots & x_3^n \\
\vdots & \vdots & \vdots & \ddots & \vdots \\
1 & x_m & x_m^2 & \cdots & x_m^n
\end{pmatrix}

\begin{pmatrix}
w_0 \\
w_1 \\
w_2 \\
\vdots \\
w_n
\end{pmatrix}
&=&
\begin{pmatrix}
1 & 1 & 1 & \cdots & 1 \\
x_1 & x_2 & x_3 & \cdots & x_m \\
x_1^2 & x_2^2 & x_3^2 & \cdots & x_m^2 \\
\vdots & \vdots & \vdots & \ddots & \vdots \\
x_1^n & x_2^n & x_3^n & \cdots & x_m^n
\end{pmatrix}

\begin{pmatrix}
y_1 \\
y_2 \\
y_3 \\
\vdots \\
y_m
\end{pmatrix}

\end{eqnarray}
```

Here, define:

```math
\boldsymbol{X} = \begin{pmatrix}
1 & x_1 & x_1^2 & \cdots & x_1^n \\
1 & x_2 & x_2^2 & \cdots & x_2^n \\
1 & x_3 & x_3^2 & \cdots & x_3^n \\
\vdots & \vdots & \vdots & \ddots & \vdots \\
1 & x_m & x_m^2 & \cdots & x_m^n
\end{pmatrix}
,
\boldsymbol{w} = \begin{pmatrix}
w_0 \\
w_1 \\
w_2 \\
\vdots \\
w_n
\end{pmatrix}
,
\boldsymbol{y} = \begin{pmatrix}
y_1 \\
y_2 \\
y_3 \\
\vdots \\
y_m
\end{pmatrix}
```

Then:

```math
\begin{eqnarray}
\boldsymbol{X}^T \boldsymbol{X} \boldsymbol{w} &=& \boldsymbol{X}^T \boldsymbol{y} \\
\boldsymbol{w} &=& (\boldsymbol{X}^T \boldsymbol{X})^{-1} \boldsymbol{X}^T \boldsymbol{y}\,...(9)
\end{eqnarray}
```

Eq. (9) is the general form of the least-squares solution (normal equation) for an n-th order function.

## Implementation

This time, we use the following packages.

```julia:Polynomial_Curve_Fitting.jl
using Distributions
using Plots
```

First, set the total number of training data points and the degree of the approximating function.

```julia:Polynomial_Curve_Fitting.jl
M = 9  # total number of training samples
N = 9  # degree of the approximating function
```

Next, define the function that generates data.

```julia:Polynomial_Curve_Fitting.jl
dist = Normal(0.0, 0.1)  # normal distribution with mean 0.0 and std 0.1
base(x) = sin.(2pi*x)  # sine wave with period 1
gen(x) = base(x) + rand(dist, length(x))  # function to generate data
```

Next, design a function that creates the following n-th order polynomial matrix X.

```math
\boldsymbol{X} = \begin{pmatrix}
1 & x_1 & x_1^2 & \cdots & x_1^n \\
1 & x_2 & x_2^2 & \cdots & x_2^n \\
1 & x_3 & x_3^2 & \cdots & x_3^n \\
\vdots & \vdots & \vdots & \ddots & \vdots \\
1 & x_m & x_m^2 & \cdots & x_m^n
\end{pmatrix}
```

```julia:Polynomial_Curve_Fitting.jl
function polynomial_matrix(x, N)
    m = length(x)
    X = fill(1.0, m)  # create a column vector of ones
    for j in 1:N
        x_vec = x .^ j  # x to the j-th power
        X = hcat(X, x_vec)  # append vector to the right
    end
    return X
end
```

Next, design a function to compute coefficient vector w of the n-th order polynomial.

```math
\boldsymbol{w} = (\boldsymbol{X}^T \boldsymbol{X})^{-1} \boldsymbol{X}^T \boldsymbol{y}
```

```julia:Polynomial_Curve_Fitting.jl
function weight(x, y, N)
    X = polynomial_matrix(x, N)
    w = (X' * X) \ X' * y
    return w
end
```

Next, design a function that outputs f(x) from estimated coefficients w and input data x.

```julia:Polynomial_Curve_Fitting.jl
function f(x, w)
    X = polynomial_matrix(x, length(w)-1)
    y = X * w
    return y
end
```
